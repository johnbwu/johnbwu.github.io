Convict Conditioning

About

Convict Conditioning is an Android fitness application I developed after being frustrated with the lack of updated companion apps on the Play Store. I felt that going on the computer to search up how to do exercises disrupted my workout flow. Convict Conditioning shows how to do each exercise with text, pictures, and videos, while tracking the user's progress. I took special care in adhering to Google's Material Design guidelines, leveraging native and custom libraries to maintain a beautiful and simple UI and UX.

SAP

About

I developed predictive maintenance software using SAP HANA and server-side JavaScript. Triggers notified the user whenever incoming sensor data was reading past set thresholds.

In addition, I worked on an oil drilling Internet of Things web application, so that offshore oil rigs could communicate with a central server while at sea.

Finally, I wrote a RESTful API so that the predictive maintenance software could create/update/delete work order and equipment data from the oil drilling web application. For example, if a certain part was sensed to be overheating, a work order would be automatically created and sent to the rig.

Technologies Used: SAPUI5 front-end JavaScript MVC framework, SAP HANA in-memory database, SQL Anywhere database, Perforce, and CSS.

Work Market

About

During my internship at Work Market, I spearheaded the full-stack development and release of the platform's forums. Working closely with product managers, I architected the data model, designed and implemented the UI, and developed features.

I implemented features such as nested commenting, post tagging, and advanced search, using the Spring MVC framework and the Apache Solr search server.

To ensure future maintainability, I also wrote an automated test suite, creating unit tests with JUnit, integration tests with Mockito, and acceptance tests with Cucumber.

Technologies Used: Java, Spring MVC, Hibernate, JavaScript, jQuery, LESS/CSS, HTML5/JSP, Apache Solr, Git, Redis, Jenkins, Amazon AWS, JUnit, Mockito, Cucumber, Groovy, JSON, Bootstrap

Find My Class!

About

I developed an native Android application with a team at the MHacks Winter 2014 Hackathon. Using the UW API, we created an app that allowed users to search when, where, and by whom courses were being taught on campus.

We scraped RateMyProfessor.com for professor scores. Users are then able to find a different class to go to if they had timing conflicts or wanted to try a different professor.

Technologies Used: Python, Java, Android, Git, Eclipse
